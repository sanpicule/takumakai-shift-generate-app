# 看護師シフト生成アプリ

## 概要

病棟看護師の月間シフト表を自動生成する Electron デスクトップアプリ。
スタッフ情報・希望休を入力し、施設ルールに従ったシフト表をマークダウンテーブル形式で出力する。

**技術スタック**：Electron / React / TypeScript / Vite / Zustand / Tailwind CSS / electron-store

---

## 施設シフトルール

- 月〜土（祝日含む）の日勤は **2名**、日曜のみ **1名**
- 当直は毎日 **1名**（2交代制）
- 当直の翌日は必ず **「明け」** にする
- 極力5連勤以内に収める（やむを得ない場合は6連勤まで許容。当直専従常勤は「当→明」サイクルでの6連勤を許可）
- 基本サイクルは「当直 → 明け → 休み」
- 月間公休数 ＝ 月の土曜数 ＋ 日曜数（祝日は平日扱い、非常勤は対象外）
- 土曜当直・日曜当直・日曜日勤には公休とは別枠で **代休** を付与
- 代休は原則当月中に取得（月末が土日の場合のみ翌月可）
- 月に1回は連休（2日以上連続の休み）を付与
- 日曜の休みはスタッフ間で均等に配分
- 勤務希望は **最優先** で反映（スタッフ間で事前調整済み）
- 人員不足日は **空欄**（他部署応援で手入力する運用）

**制約の優先順位**（満たせない場合は下位から緩和。ただし公休数は緩和禁止）：
1. 最低人員の確保
2. 公休数の確保（**緩和禁止**：必ず月の土日数を確保する）
3. 5連勤の回避
4. 当直→明→休サイクル
5. 勤務希望の反映
6. 連休の付与
7. 日曜休みの均等配分
8. 代休の当月消化

---

## 出力形式

- **マークダウンテーブル記法**（`|` 区切り）で出力する
- コードブロック（` ``` `）内への出力は禁止
- ヘッダー：`スタッフ | 1(曜) | 2(曜) | … | 公休数 | 代休数 | 日勤数 | 当直数 | 日曜休み数 | 最大連勤数 | 連休数`
- フッター：「日勤人数」行と「当直担当」行の2行
- 制約違反・緩和がある場合はテーブルの後に箇条書きで報告

---

## 完了条件

実装・修正後は以下をすべてクリアしていること。

- 必須制約（当直の空欄なし・日勤人数・明け付与）が満たされている
- 出力がマークダウンテーブル形式になっている
- 集計列7列とフッター2行が存在する
- 精度目標 **85〜90%**（制約をすべて同時に満たすことは求めない）
- **既存コードを修正・機能追加した場合は、対応するテストを修正または追加し `npm run test` が全件グリーンであること**

詳細なテスト仕様・チェックリストは `docs/tests/` を参照すること。

---

## 禁止事項

- テスト結果を合格に見せるためにルール・要件を変更しない
- シフト生成後に遡ってルールを変更してから再評価しない
- 仕様に書かれていない事項を推測で補わない（不明点はユーザーに確認）
- APIキー・シークレットをコードにハードコードしない
- `.env` ファイルをコミットしない
