# テスト仕様・チェックリスト（統合テスト）

> ユニットテストの仕様は `docs/tests/unit-test/` を参照すること。
> - `engine.md` — シフト生成エンジン（generator.ts）
> - `output.md` — 出力フォーマット生成（generateMarkdownTable / generateCSV）
> - `store.md` — 状態管理（useAppStore）

## 合格基準

### 必須制約（1件でも違反 → 不合格）

| ID | 確認内容 |
|----|---------|
| D1 | 当直担当が全日付に1名いる（空欄なし） |
| D2 | 月〜土（祝日含む）の日勤人数が2名 |
| D3 | 日曜の日勤人数が1名 |
| D4 | 同日に当直が2名以上いない |
| D5 | 全当直の翌日が「明」になっている |

### 出力形式（1件でも違反 → 不合格）

| ID | 確認内容 |
|----|---------|
| H1 | マークダウンテーブル記法（`\|` 区切り）で出力されている |
| H2 | コードブロック（` ``` `）内に出力されていない |
| H3 | ヘッダー行が「スタッフ \| 1(曜) \| 2(曜) … \| 集計7列」形式 |
| H4 | フッター行に「日勤人数」「当直担当」の2行がある |
| H5 | 集計列7列がすべて含まれている（公休数/代休数/日勤数/当直数/日曜休み数/最大連勤数/連休数） |
| H6 | 制約違反・緩和内容がテーブルの後に箇条書きで報告されている（違反がある場合） |
| H7 | ヘッダー行の曜日表示が実際のカレンダーと一致している |

### 重要確認（緩和報告があれば条件付き合格）

| ID | 確認内容 |
|----|---------|
| E1 | スタッフ（非常勤除く）の公休数 = 月の土日数（**緩和禁止・必須**） |
| E2 | 土曜当直に代休がついている |
| E3 | 日曜当直に代休がついている |
| E4 | 日曜日勤に代休がついている |

| C6 | 当直専従スタッフ（非常勤）に「日」がない |
| C7 | 非常勤スタッフがユーザー指定日に当直している |

### 品質確認（調整制約）

|----|---------|

## テスト実施ルール

- 成果物は `docs/tests/results/<テストID>_<概要>/` に格納する
  - `シフト.md` — 生成したシフト表
  - `テスト結果.md` — チェックリスト記入済みのテスト結果
- テストが「NG」でも合格に見せるための修正・調整は行わない
- 生成と評価は別ステップとして実施する（生成後にルールを変更してから再評価しない）
